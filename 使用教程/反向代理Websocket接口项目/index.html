反向代理Websocket接口项目

关于WebSocket接口支持的注意事项： 1 上游服务器中，如果有WebSocket接口，不需要在"proxy_pass"项中进行特别额外的配置；

2 配置后，可以通过：ws://api.quwancode.com/XXXX; 访问到上游服务器中相应的WebSocket接口；

一 需要代理的目标项目​

以一个具体的包含Websocket接口的Streamlit项目为例： https://github.com/JuneWaySue/my_streamlit_app

下载clone到本地后； 先安装依赖包： pip install -r requirements.txt 然后运行程序：streamlit run my_streamlit.py

可以看到，Web服务运行在：http://localhost:8501

二 配置Fast-Https代理​

下面，将这个项目通过Fast-Https服务器代理到通过域名，并在80端口访问；

假定我们要使用的域名为：www.abc.com； 请在host配置中，加入： 127.0.0.1 www.abc.com

具体配置文件fast-https.json可以参考：

{
  "log_root":"./logs",
  "pid": "./fast-https.pid",
  "http": {
    "include": "mime.types",
    "server": [      
      {
        "listen": 80,
        "server_name": "www.abc.com",
        "location": [
          {
            "url": "/",
            "type": "proxy",
            "proxy_pass": "http://127.0.0.1:8501",
            "proxy_set_header" : [
              {
                "HeaderKey": "Host", 
                "HeaderValue": "$host"
              }
            ]
          }

        ]
      }
    ]
  }
}

然后在浏览器中，通过输入域名访问：http://www.abc.com即可；
