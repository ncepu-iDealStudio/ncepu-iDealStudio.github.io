配置发布本地静态站点

1 软件获取​

打开系统软件发行下载地址：https://gitee.com/ncepu-bj/fast-https/releases；

确定您的静态站点发布部署的平台，如果是Windows平台，请下载相应的Windows的版本（windows平台大部分情况下可以下载这个文件即可：fast-https_Windows_x86_64.zip）； ** **

如果是linux平台，请下载与服务器计算架构相匹配的版本；如果是普通X86架构服务器，请下载这个版本：fast-https_Linux_x86_64.tar.gz

2 安装部署-windows平台​

软件包解压​

下面以在windows平台发布一个静态网站为例，来说明Fast Https部署本地静态网站的过程；下载后的软件如下：

对压缩包解压，解压后的文件如下：

可以将解压后的软件包，拷贝到合适的目录下；假定为：D:\fast-https_Windows_x86_64

静态站点代码上传到服务器​

在文件夹：D:\fast-https_Windows_x86_64\httpdoc中，创建站点文件夹，文件夹名称可以为：www.myweb.com；将您准备发布的静态站点代码，拷贝到文件夹：D:\fast-https_Windows_x86_64\httpdoc\www.myweb.com下；

测试用静态站点资源包：📎theme.zip

修改配置文件​

在文件夹：D:\fast-https_Windows_x86_64\config中，找到服务器主配置文件：fast-https.json，打开文件进行相应的配置：

**具体配置文件： **

{
  "log_root":"./logs",
  "pid": "./fast-https.pid",
  "http": {
    "include": "mime.types",
    "server": [
      {
        "listen": 8080,
        "server_name": "localhost",
        
        "location": [
          {
            "url": "/",
            "type": "local",
            "root": "./httpdoc/www.myweb.com",
            "index":["index.html", "index.htm"]
          }

        ]
      },
      {
        "listen": "443 ssl",
        "server_name": "localhost",
        "ssl_certificate": "config/cert/localhost.pem",
        "ssl_certificate_key": "config/cert/localhost-key.pem",
        "location": [
          {
            "url": "/",
            "type": "local",
            "root": "./httpdoc/www.myweb.com",
            "index":["index.html", "index.htm"] 
          }
        ]
      }
    ]
  }
}

启动服务器管理程序​

在服务器目录D:\fast-https_Windows_x86_64中，找到打开monitor.exe

启动Fast-Https服务，点击任务栏右下角托盘：

之后在浏览器访问http://localhost8080 即可看到访问页面：

3 Lnux平台安装部署​

3.1 静态站点代码上传服务器​

在文件夹：/usr/local/fast-https/httpdoc/ 中，创建站点文件夹，文件夹名称可以为：www.myweb.com；将您准备发布的静态站点代码，拷贝到文件夹：/usr/local/fast-https/httpdoc/www.myweb.com下；

测试用静态站点资源包：📎theme.zip

3.2 修改配置文件​

$ vi /usr/local/fast-https/config/fast-https.json

按照以下方式修改配置文件：

{
  "log_root":"./logs",
  "pid": "./fast-https.pid",
  "http": {
    "include": "mime.types",
    "server": [
      {
        "listen": 8080,
        "server_name": "localhost",
        
        "location": [
          {
            "url": "/",
            "type": "local",
            "root": "./httpdoc/www.myweb.com",
            "index":["index.html", "index.htm"]
          }

        ]
      },
      {
        "listen": "443 ssl",
        "server_name": "localhost",
        "ssl_certificate": "config/cert/localhost.pem",
        "ssl_certificate_key": "config/cert/localhost-key.pem",
        "location": [
          {
            "url": "/",
            "type": "local",
            "root": "./httpdoc/www.myweb.com",
            "index":["index.html", "index.htm"] 
          }
        ]
      }
    ]
  }
}

若有收获，就点个赞吧

3.3 启动并测试​

$ ./fast-https

备注： 上面这种启动方式，当用户退出终端界面，或者退出监听时，服务程序会自动退出终止；如果在生产环境下启动，可以考虑后台进程启动方式：

$ nohup ./fast-https &

在浏览器中输入： 浏览器访问http://ip:8080/index.html，即可看到部署的静态网站效果；
